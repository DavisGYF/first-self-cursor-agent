# 国内部署（无需科学上网）

若你**只考虑大陆访问**，建议**不要依赖 Railway / Render 等境外 PaaS**：它们的**控制台网站**在国内常不稳定或需代理，运维也不方便。

下面三种方式控制台与机器都在**国内厂商**，一般**无需 VPN**。

### 要钱么？有没有免费 / 省钱？

| 情况 | 说明 |
|------|------|
| **长期完全免费、又稳定跑 Node + 公网访问** | 国内**基本没有**这种「一直白嫖的 VPS」；大模型调用本身也要 **API 费用**（OpenAI/国内模型按量计费）。 |
| **免费试用** | 阿里云 / 腾讯云等常给**新用户试用金或几天试用**（政策会变，以官网为准），适合练手，到期后要付费或停服。 |
| **最省钱、最常见** | **轻量应用服务器**新用户/活动价，常见 **几十～一百多元/年**（配置低、个人演示够用）；学生机、秒杀活动可关注厂商首页。 |
| **一分钱不花、只自己玩** | 继续 **本机 `npm run dev`**，或用 **内网穿透**（如 ngrok 国内替代、花生壳等）临时把本机暴露出去——**不适合**正经给别人长期用，且电脑要开着。 |

**结论**：想**稳定给国内别人访问**，一般至少要有**一台最低配国内云主机**的预算；想**最省**，就盯**新用户首单 / 活动价轻量**，别指望长期完全免费云主机。

---

## 方案 A：腾讯云 / 阿里云「轻量应用服务器」（最通用）

适合：**一台小 VPS + SSH**，和「买台国内云主机」一样，全中文控制台。

**走阿里云、第一次部署**：请直接打开 **[DEPLOY-ALIYUN.md](./DEPLOY-ALIYUN.md)**（从买机器到 `pm2` 全流程）。

1. 在 [腾讯云轻量](https://cloud.tencent.com/product/lighthouse) 或 [阿里云轻量](https://www.aliyun.com/product/swas) 买一台 **Linux**（Ubuntu 22.04 即可），选**国内地域**。
2. 安全组放行 **3000**（或你后面用的端口）。
3. SSH 登录后安装 Node **20**（可用 [nvm](https://github.com/nvm-sh/nvm) 或厂商镜像自带）。
4. 拉代码（**Gitee** 或 **Coding** 托管在国内访问更稳；也可用 GitHub）：

   ```bash
   git clone <你的仓库地址>
   cd <项目目录>
   npm ci && npm run build
   ```

5. 配置环境变量后启动（二选一）：

   ```bash
   export OPENAI_API_KEY=你的key
   npm start
   ```

   常驻推荐 **PM2**：`npm i -g pm2 && pm2 start npm --name ai-copilot -- start`

6. 浏览器访问：`http://服务器公网IP:3000`。要做 **HTTPS + 域名** 再在机器上装 **Nginx** 配证书；**备案**按厂商要求（仅当使用未备案域名指向大陆服务器时）。

**特点**：完全自主、国内访问快；缺点是要自己管一点 Linux。

---

## 方案 B：腾讯云「云托管」等容器服务（偏可视化）

适合：希望**在网页里点部署**、用**镜像**交付。

1. 本地或 CI 构建镜像（仓库根目录已提供 **`Dockerfile`**）：

   ```bash
   docker build -t ai-copilot:1 .
   ```

2. 把镜像推到**腾讯云容器镜像服务**（或阿里云 ACR），在 **[云托管](https://cloud.tencent.com/product/tcb)**（或同类）里创建服务，选该镜像，端口 **3000**，环境变量里填 **`OPENAI_API_KEY`**，开启公网访问。
3. 控制台会分配访问地址（具体以腾讯云文档为准）。

**特点**：控制台在国内；需要会一点 Docker / 镜像推送。

---

## 方案 C：Railway（仅适合能稳定访问境外控制台时）

仓库里保留的 **`railway.toml` / `package.json` 根脚本** 仍可用于 Railway；若你在国内**打不开** [railway.com](https://railway.com)，请改用上面 **A 或 B**。

---

## 环境变量（与本地 `server/.env` 一致）

| 变量 | 说明 |
|------|------|
| `OPENAI_API_KEY` | 必填 |
| `OPENAI_BASE_URL` | 可选 |
| `DEFAULT_MODEL` | 可选 |
| `PORT` | 可选，默认 3000；云托管请与容器暴露端口一致 |
| `DATABASE_PATH` | 可选；需要会话/RAG 长期落盘时，指到**持久卷**上的路径 |

---

## 小结

| 需求 | 建议 |
|------|------|
| 国内访问、不想折腾境外网站 | **轻量服务器 + PM2**（方案 A） |
| 想要容器 + 国内控制台 | **Dockerfile + 腾讯云云托管**（方案 B） |
| 能打开 Railway 且接受境外服务 | 见 [RAILWAY.md](./RAILWAY.md) |
