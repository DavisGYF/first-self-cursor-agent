# 阿里云轻量：第一次部署（前端 + 后台一起）

> **你选了 Docker 应用镜像**：请直接看 **[DEPLOY-ALIYUN-DOCKER.md](./DEPLOY-ALIYUN-DOCKER.md)**（`docker compose` 一步步）。  
> 本文是 **Ubuntu 裸机 + Node + PM2** 路线，与 Docker **二选一**，不要两套同时占 **3000** 端口。

适合：**已有 / 能注册阿里云账号**，愿意买一台**国内轻量服务器**（通常几十～一百多/年，新用户更便宜）。

下面分成两块：

- **我在仓库里已帮你准备好的**：根目录 `npm run build` / `npm start`、后端托管 `web/dist` 等，你**不用改代码**。
- **必须你自己在阿里云网页上点的**：注册、付钱、开防火墙、拿密码——**别人替不了你**，我按顺序写清楚。

---

## 〇、你先准备这三样

| 东西 | 说明 |
|------|------|
| **阿里云账号** | 你说已有；没有就去 [aliyun.com](https://www.aliyun.com) 注册并**实名认证**（买国内服务器一般要实名）。 |
| **支付方式** | 支付宝/微信等能付款即可。 |
| **大模型 API Key** | 例如 OpenAI 或兼容网关的 Key，部署时要写到服务器环境变量里（下面有命令）。 |
| **代码在 Git 上** | 服务器上要 `git clone`。代码在 **GitHub** 或 **Gitee** 都行；私有仓库需要配置 SSH 或 Token（第一次建议用 **Gitee 公开仓库** 或 **GitHub 公开仓库**，最省事）。 |

## 一、在阿里云买「轻量应用服务器」（你要自己操作）

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)（若找不到，首页搜「**轻量应用服务器**」）。
2. 进入 **轻量应用服务器** → **创建服务器** / **购买**。
3. 选择建议（按默认便宜的选即可）：
   - **地域**：离你或你的用户近的国内节点（如 **华东 / 华北**）。
   - **镜像**：选 **Ubuntu**（建议 **22.04**），不要选 Windows（除非你很熟）。
   - **套餐**：选**最低配**一般够用（1 核 1G～2G，个人 Demo 可以）。
4. 设置 **root 密码**（或选密钥对；第一次用**密码**简单）：**务必记在小本本**，待会 SSH 要用。
5. 付款，等实例状态变为 **运行中**。

> 具体按钮名称以阿里云当前页面为准，认准「轻量」不是「ECS」也行，轻量对新手更省事。

---

## 二、放行端口（你要自己操作，否则浏览器打不开）

你的服务默认监听 **3000** 端口，必须在**阿里云防火墙**里放行：

1. 打开 **轻量应用服务器** → 点进你的那台实例。
2. 找到 **防火墙** / **安全** / **规则** 一类菜单。
3. **添加规则**：协议 **TCP**，端口 **3000**，来源 **0.0.0.0/0**（允许所有人访问；以后若要安全可再改）。
4. 保存。

---

## 三、拿到公网 IP（你要自己看一眼）

在实例详情页可以看到 **公网 IP**，形如 `47.xxx.xxx.xxx`。下面用 `你的公网IP` 代替。

---

## 四、在你自己的电脑上 SSH 登录（Mac 自带终端）

打开 **终端**（Terminal），执行（把 IP 换成你的）：

```bash
ssh root@你的公网IP
```

第一次会问 `yes/no`，输入 `yes`；然后输入你买机器时设的 **root 密码**（输入时**不显示字符**，正常）。

登录成功后，提示符会变成类似 `root@xxx:~#`，说明已经在云服务器里了。

---

## 五、在服务器上：安装依赖（复制粘贴即可）

以下命令**整段在服务器里执行**（一行行复制也行）。

### 1）更新系统并装基础工具

```bash
apt update && apt install -y git curl ca-certificates build-essential
```

`build-essential` 用来编译 `better-sqlite3`，**必须装**。

### 2）安装 Node.js 20（用 nvm，避免系统自带 Node 太老）

```bash
export NVM_DIR="$HOME/.nvm"
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
# 重新加载 nvm（或关掉 SSH 再连一次）
. "$NVM_DIR/nvm.sh"
nvm install 20
nvm use 20
node -v   # 应显示 v20.x
```

若 `curl` 访问 GitHub 很慢，可换国内镜像装 nvm（略麻烦）；多数情况下能装上。

### 3）安装 PM2（让服务崩溃后自动拉起、关机重启可恢复）

```bash
npm install -g pm2
```

### 4）拉代码（把地址换成你自己的仓库）

**公开仓库示例：**

```bash
cd /root
git clone https://github.com/你的用户名/你的仓库名.git
cd 你的仓库名
```

若是 **Gitee**，把上面的 `https://gitee.com/...` 换成你的地址。

**私有仓库**：需要在服务器上配 SSH 公钥或 HTTPS Token，第一次部署建议先把仓库**公开**，或减少折腾。

### 5）安装依赖并构建（仓库根目录已有脚本）

一定要在**项目根目录**（有根目录 `package.json` 的那一层）：

```bash
npm ci && npm run build
```

这一步会：构建前端 `web/dist`、安装 `server` 依赖，可能要 **几分钟**。

### 6）配置环境变量（二选一）

**方式 A：在仓库根目录创建 `.env`（推荐）**

`dotenv` 会从**当前工作目录**加载，在**项目根目录**执行时，在**根目录**放 `.env`：

```bash
cd /root/你的仓库名   # 确保在根目录
nano .env
```

写入（把 key 换成你的，保存：`Ctrl+O` 回车，`Ctrl+X` 退出）：

```env
OPENAI_API_KEY=你的key
```

**方式 B：只在当前会话有效**

```bash
export OPENAI_API_KEY=你的key
```

### 7）用 PM2 启动（必须在仓库根目录）

```bash
cd /root/你的仓库名
pm2 start npm --name ai-copilot -- start
pm2 save
pm2 startup
```

`pm2 startup` 会打印一行以 `sudo` 开头的命令，**按提示复制执行一次**，这样服务器重启后 PM2 会自动起来。

---

## 六、浏览器访问

在浏览器打开：

```text
http://你的公网IP:3000
```

若打不开，检查：

1. 阿里云防火墙是否放行 **3000**；
2. `pm2 status` 里 `ai-copilot` 是否 **online**；
3. `pm2 logs ai-copilot` 里有没有报错（缺 Key、端口占用等）。

---

## 七、以后更新代码怎么做

在你**自己电脑**上改完代码并 `git push` 后，到服务器：

```bash
cd /root/你的仓库名
git pull
npm ci && npm run build
pm2 restart ai-copilot
```

## 九、常见问题（第一次必看）

| 现象 | 可能原因 |
|------|----------|
| SSH 连不上 | 安全组/防火墙没放行 **22**；或 IP 填错；或密码错。 |
| 网页打不开 | 没放行 **3000**；或 PM2 没启动；或服务报错。 |
| 构建报错 `better-sqlite3` | 没装 `build-essential`，回到第五节 1）执行 `apt install -y build-essential` 再 `npm run build`。 |
| 对话报错缺 Key | `.env` 不在**仓库根目录**，或变量名不是 `OPENAI_API_KEY`。 |

## 十、域名和备案（以后再说）

- 只用 **IP:3000** 访问，**不用备案**。
- 若要用 **自己的域名 + 80/443**，通常需要 **备案** 和 **Nginx 反代**，这一步等你跑通 IP 后再做即可。
